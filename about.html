StuMeeについて

StuMeeは「Study Meeting」の略であり、学習者同士の交流の場や質問をしやすい環境を 設けることで、学習の敷居を低くし、挫折する学習者を減らすことを目標としています。

StuMeeの特徴

フレームワーク

StuMeeはPythonのフルスタックフレームワークであるDjangoを用いて作成されています。

ライブラリ

ライブラリ一覧
Django Pythonのフルスタックフレームワーク。大体これで作ってる。
psycopg2 PostgresとDjangoをつなぐために必要。
social-auth-app-django Google oauthを簡単に利用するために必要。
django-taggit タグ周りを全部やってくれる。便利。
Pillow Pythonの画像処理ライブラリ。便利。これがないとDjangoで画像は扱えない。
django-imagekit 勝手に画像のサイズや種類を加工してくれる。便利。
django-cleanup 画像やファイルなどを変更した時に残るゴミを削除してくれる。
Django channels Web socketを利用したチャットアプリの開発が容易に
channels_redis Django channelsがredisを認識できるようにする為に必要
django-storages minIOをストレージとして利用するために必要。
boto3 上記と同様。

このほかに、bootstrap4やjqueryを用いて実装しています。

機能の説明

ログイン周り
Google Oauthを利用した、Googleアカウントでログインできる機能を実装。
ログアウト処理はDjangoのデフォルト機能を利用。

Meeting
掲示板のようなものを作成。ユーザは誰でもスレッドを作成でき、コメント、いいね！ができる。
スレッドはいいね順や日付順などで並び替えられ、コメントはいいね順に並び替えられる。
TA以上の権限を持つものであれば、スレッドをPickすることができ、Pickされたスレッドは 一番上に並ぶようになる。
いいね機能はリダイレクトではなく、Ajaxを用いて非同期処理を行っている。

Setting
この画面でユーザは自信の権限及び名前、アイコン画像を変更することができる。

Study
Teacherの権限を持つユーザのみが、コースの設立及びカテゴリの追加操作を行うことができる。
また、そのコースの設立者のみがコースの管理や削除を行うことができる。

検索まわり
Studyページ内及びMeetingページ内を検索することができる。
この二つの検索フォームは独立している。

手順書(自分用)

本番環境のDocker image更新時
1. ローカルのsetting.pyにてDebug = Falseに設定。
2. ローカルの方で、docker-compose run web ./manage.py collectstaticを実行し、動作を確認
3. ローカルのsetting.pyにてEndpoint URLを本番環境のminIOのアドレスを指すように変更。
4. ローカルにてdocker-compose downを実行し、稼働中のコンテナを終了
5. ローカルにてdocker-compose buildを実行し、コードをimage内に含めるようにする
6. ローカルにてdocker-compose up -dを実行し、コンテナを作成。
7. ローカルにて、docker commit を実行し、Docekr Hubにpushする用のimageを作成
8. ローカルにてdocker push ryuusan/stumee_web:stableを実行し、7で作成したimageをpushする。
9. sshを用いて本番環境にアクセスする。
10. stumeeディレクトリに移動。
11. docker-compose.ymlファイルがあることを確認し、docker pull ryuusan/stumee_web:stableを実行。
12. docker-compose up -dコマンドを実行する。
13. docker-compose startコマンドを実行し、サービスを稼働させる。
14. docker-compose run web ./manage.py migrateを実行する。
15. docker-compose run web ./manage.py collectstaticを実行する。
16. サービスにアクセスして動作を確認。
